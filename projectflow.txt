1. create the repo and clone in the local
2. create the code for generating the sample csv file
3. Now add the code and push over the generating
4. initialzed the DVC install dvc using the pip command creat the requirement file
    - if there is proble that dvc version is not showing up when typing dvc --version install the setup using chocaltey
    - also check your environment path for the DVC
    - do this project level but i recommend setup at global level
5. dvc init will do two things
    - .dvc folder
    - .dvcignore same like .gitignore file to ignore the files
6. <right now we are using the local setup using the MS Azure right now focus on the local setup>
7.  mkdir azure to create the folder in the project
8. Now we want to track the azure folder using the command dvc remote add -d myremote azure
    - documentation "https://doc.dvc.org/command-reference/remote/default"
9. now tell dvc to add folder data so that any commit changes will be notified to dvc use command dvc add data/
10. Error happen because the git is already tracking the folder
    - git rm -r --cached 'data'
    - git commit -m "stop tracking data"
11. Now add the data which dvc add data/ 
    - this will create the data.dvc which has unique hash id
12. check the dvc status
